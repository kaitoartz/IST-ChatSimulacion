# Plantilla de Definición de Tarea para Gemini CLI

Usa este formato para describir tus tareas. Cuanto más específica sea la información, mejor podré ayudarte.

## 1. Objetivo Principal
*Describe en una frase qué quieres conseguir.*
> Ejemplo: Implementar un sistema de login con validación de dos factores.

## 2. Contexto
*Explica brevemente por qué es necesaria esta tarea y en qué parte del proyecto encaja.*
> Ejemplo: Los usuarios necesitan una forma segura de acceder a la plataforma. Esto afecta al módulo de autenticación en `src/auth`.

## 3. Archivos Afectados (Opcional)
*Lista los archivos o directorios clave si los conoces.*
- `ruta/al/archivo.ts`
- `src/carpeta_importante/`

## 4. Requisitos Específicos
*Detalla las reglas, tecnologías o restricciones.*
- [ ] Debe usar [Tecnología/Librería]
- [ ] Debe seguir el patrón de diseño [Nombre del Patrón]
- [ ] No modificar [Archivo/Función protegida]

## 5. Instrucciones Paso a Paso (Opcional)
*Si tienes una idea clara de cómo hacerlo, numérala aquí.*
1. Crear la función X...
2. Actualizar el test Y...

## 6. Criterios de Aceptación / Resultado Esperado
*¿Cómo sabremos que la tarea está terminada y bien hecha?*
- El código compila sin errores.
- Los tests pasan (o se han añadido nuevos tests).
- La funcionalidad X se comporta de la manera Y.

---
**Nota:** Puedes borrar las secciones que no necesites para tareas simples.
